ZCrud vs jTable

The main idea of ZCrud is heavily inspired by jTable, although implementation is very different. There are with some major differences:

  • The most important change is to build all HTML code of forms using a template engine (ZPT-JS).
  • ZCrud does not depend on any big javascript API (like Jquery UI).
  • ZCrud does not create any dialog form, only simple HTML forms.

Common features

ZCrud and jTable share a lot of features. Some of them are:

  • Automatically creates HTML table and loads records from server.
  • Creates some typical forms:
    • Create record forms. When user creates a record, it sends data to server and adds the same record to the table in the page.
    • Edit record forms. When user edits a record, it updates server and updates all cells on the table in the page.
    • Delete record forms. When user deletes a record, it deletes the record from server and delete the record from the table in the page.
  • All styling of table and forms are defined in a CSS file, so you can easily change style of everything. CSS file is well defined and commented.
  • It comes with pre-defined color themes.
  • Supports server side paging, sorting and filtering.
  • Allows user to select rows.
  • It is not depended on any server side technology.
  • It can be localized for non supported languages easily.

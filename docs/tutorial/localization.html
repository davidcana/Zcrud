Introduction

ZCrud includes full support for i18n and l10n. All texts, date and number formats are stored into i18n files. Zcrud includes support for english and spanish, but it is easy to support other languages.

The i18n files

For supporting a language is needed:

  1. A JSON file with the texts and configuration options used by ZCrud. It includes date formats, number formats, days of week, months, texts in buttons...
  2. A JSON file per entity for implementing CRUD. It includes fields names, field descriptions, texts in customized buttons...

The i18n directory contains these files:

  • en-common.json. English texts and configuration options used by ZCrud.
  • en-people.json. English texts and configuration options used by ZCrud in examples.
  • es-common.json. Spanish texts and configuration options used by ZCrud.
  • es-people.json. Spanish texts and configuration options used by ZCrud in examples.

Contents of common files

Let's see the contents of a common file:

  • /CONF/. Language and locale setting.
  • _general. General settings (date, time and number formats).
  • _weekDays. Names of week days.
  • _months. Names of months.
  • _list. Texts used in lists.
  • _form. Texts used in forms.
  • _buttons. Texts used in buttons.
  • _validation. Texts used in validation.

Contents of entity files

Let's see the contents of an entity file:

  • /CONF/. Language and locale setting.
  • _general. General settings (entity id).
  • _fields. Names and descriptions of fields. You must add 2 entries of this type per field, one for title (with _title suffix) and one for description (with _description suffix). So if you add a field with years as id, you must add 2 entries:
    • years_title. It will be use as a title for this field in lists and forms.
    • years_description. It will be use as a description (only in forms).
  • _options. Texts of options in several types of HTML fields (checkboxes, dataList, select and radio). You must add 1 entry of this type per option (with _option suffix).
  • _buttons. Texts used in buttons.

Configuring options about i18n

Default configuration is: 

    i18n: {
        language: 'en',
        filesPath: '/i18n/',
        files: { 
            en: [ 'en-common.json' ],
            es: [ 'es-common.json' ] 
        }
    },

To support people our configuration is: 

    i18n: {
        language: 'en',
        files: { 
            en: [ 'en-common.json', 'en-people.json' ],
            es: [ 'es-common.json', 'es-people.json' ] 
        }
    },

If we add support to another language (for example french) we must to write fr-common.json and fr-people.json files. Our configuration would be: 

    i18n: {
        language: 'fr',
        files: { 
            en: [ 'en-common.json', 'en-people.json' ],
            es: [ 'es-common.json', 'es-people.json' ],
            fr: [ 'fr-common.json', 'fr-people.json' ]
        }
    },

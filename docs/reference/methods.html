ZCrud API reference - Methods

ZCrud exposes some methods to make it easy some customizations:

  • addRecord( data ) -
    Add a new record to the table. Possible arguments:
    • data.record (object, required). The record object that will be added to the table.
    • data.clientOnly (boolean, optional, default: false). If this is true, ZCrud does not add record to the server, only adds to the table.
    • data.url (string, optional, default: url property of page). You can supply different URL to create the new record on the server.
    • data.success (function, optional, default: undefined). A callback function that can be used to be notified when record is successfully added to the table.
    • data.error (function, optional, default: undefined). A callback function that can be used to be notified if any error occurs during adding of the new record.
  • deleteRecord( data ) -
    Delete an existing record from the table. Possible arguments:
    • data.key (string, required). The id of the record to delete.
    • data.clientOnly (boolean, optional, default: false). If this is true, ZCrud does not add record to the server, only adds to the table.
    • data.url (string, optional, default: url property of page). You can supply different URL to create the new record on the server.
    • data.success (function, optional, default: undefined). A callback function that can be used to be notified when record is successfully added to the table.
    • data.error (function, optional, default: undefined). A callback function that can be used to be notified if any error occurs during adding of the new record.
  • deselectRecords( records ) -
    Deselect one or more rows on the table depending on the supplied records. It uses the kay of each record to define the rows to Possible arguments:
    • records (array of objects, required). Defines the rows to deselect.
  • deselectRows( rows ) -
    Deselect one or more rows on the table. Possible arguments:
    • rows (jQuery selection, required). The rows to deselect.
  • destroy( ) -
    Completely removes table from its container. No arguments.
  • getRecordByKey( key ) object
    Get record by key field's value of the record. Possible arguments:
    • key (string, required). The id of the record.
  • getRecords( ) list of objects
    Get all visible records in the list. No arguments.
  • getRowByKey( key ) jQuery selection object
    This method is used to get related row (jQuery selection) of a record by key field's value of the record. Possible arguments:
    • key (string, required). The id of the record.
  • getSelectedRecords( ) array of objects
    Return an array with the selected records. No arguments.
  • getSelectedRows( ) jQuery selection object
    Return a jQuery object with the selected rows. No arguments.
  • init( options, callback ) -
    Set options and normalize them. Possible arguments:
    • options (object, required). A full configuration options object for ZCrud.
    • callback (function, optional, default: undefined). A callback function that will be executed after all asynchronous code.
  • renderList( data, callback ) -
    Get data from server (if needed) and render the list page. Possible arguments:
    • data.filter (object, optional, default: undefined). A record used as a filter.
    • data.load (boolean, optional, default: true). Defines whether ZCrud must get the records from server or not (then use an empty array).
    • data.records (array of objects, optional, default: undefined). Defines the records to show, so no server request is done.
    • callback (function, optional, default: undefined). A callback function that will be executed after all asynchronous code.
  • selectRecords( records ) -
    Select one or more rows on the table depending on the supplied records. It uses the kay of each record to define the rows to Possible arguments:
    • records (array of objects, required). Defines the rows to select.
  • selectRows( rows ) -
    Select one or more rows on the table. Possible arguments:
    • rows (jQuery selection, required). The rows to select.
  • showCreateForm( ) -
    Open a 'create new record' form dialog. No arguments.
  • showDeleteForm( key ) -
    Open a 'delete record' form dialog. Possible arguments:
    • key (string, required). The id of the record to delete.
  • showUpdateForm( key ) -
    Open a 'update record' form dialog. Possible arguments:
    • key (string, required). The id of the record to update.
  • updateRecord( data ) -
    Update an existing record on the table. Possible arguments:
    • data.key (string, required). The id of the record to update.
    • data.record (object, required). The record object that will be updated.
    • data.clientOnly (boolean, optional, default: false). If this is true, ZCrud does not send the record to the server, only updates the table.
    • data.url (string, optional, default: url property of page). You can supply different URL to update the record on the server.
    • data.success (function, optional, default: undefined). A callback function that can be used to be notified when record is successfully updated.
    • data.error (function, optional, default: undefined). A callback function that can be used to be notified if any error occurs during updating the record.

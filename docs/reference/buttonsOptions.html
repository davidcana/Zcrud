ZCrud API reference - Buttons - Configuration options for buttons

Button options

The list of configuration options of buttons is:

  • bindableIn object {}
    It defines the bindable targets. An example: 
        bindableIn: {
        listRow: true
    }
    This allows to bind the button to the listRow (rows in list pages).
  • notUseInPages Array of string []
    Makes it easy to not use a button in some types of pafes. For example: 
        notUseInPages: [ 
        'delete' 
    ]
    This will make ZCrud does not use this button into delete pages.
  • cssClass String undefined
    The CSS class of this button. If not set ZCrud will use the type property.
  • run function undefined
    A function binded to the click event of the button. An example: 
        run: function( event ){

        event.preventDefault();
        event.stopPropagation();

        ++runCounter;
    }
  • textsBundle object undefined
    Defines the title and the content texts. For example: 
        textsBundle: {
        title: undefined,
        content: {
            translate: false,
            text: 'Copy original members'
        }  
    }
    This defines 'Copy original members' (without translating it) as the content of the button and no text as the title.
  • type String undefined
    The type of the button. For example 'undo', 'redo', 'form_cancel', 'form_submit', ...
